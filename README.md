# Emotion Analysis for African Languages

**COS 760 — Group 12 · University of Pretoria · S1 2026**

A comparative study of TF-IDF and multilingual transformer models (mBERT, XLM-R)
for **multi-label emotion classification** in three Bantu languages using the
[BRIGHTER dataset](https://arxiv.org/abs/2502.11926).

---

## Research Questions

1. How do multilingual transformer models compare to a TF-IDF baseline for multi-label emotion classification in Bantu languages?
2. How does performance vary across isiZulu, isiXhosa, and Swahili, and what role does training data volume play?
3. What common prediction errors arise, and what do they reveal about class imbalance and model limitations?

---

## Key Results

| Model | isiZulu (F1-mac) | isiXhosa (F1-mac) | Swahili (F1-mac) |
|---|---|---|---|
| TF-IDF + Logistic Regression | **0.257** | **0.265** | **0.250** |
| mBERT (fine-tuned) | 0.177 | 0.240 | 0.193 |
| XLM-R (fine-tuned) | 0.135 | 0.222 | 0.223 |

> The TF-IDF baseline outperforms both transformers on macro F1 under these data constraints.
> Transformers achieve substantially higher recall (0.53–0.77 vs 0.29–0.32 baseline)
> thanks to per-label positive-class weighting (`pos_weight`).

Full results and discussion: [`documentation/Group12_Report.pdf`](documentation/Group12_Report.pdf)

## Running Experiments

All experiments are controlled from a single file. Open `main.py` and set the
phase toggles at the top:

```python
RUN_WEEK2 = True    # Baseline + exploratory analysis
RUN_WEEK3 = False   # mBERT + XLM-R fine-tuning (slow — CPU only)
RUN_WEEK4 = False   # Cross-lingual evaluation + final summary
```

Then run:

```bash
python main.py
```

### Run all phases

```python
RUN_WEEK2 = True
RUN_WEEK3 = True
RUN_WEEK4 = True
```

### Transformer fine-tuning (offline mode)

The transformer models (`bert-base-multilingual-cased`, `xlm-roberta-base`)
must be cached locally before running Week 3. Once cached, use offline mode
to avoid network calls:

```bash
HF_HUB_OFFLINE=1 TRANSFORMERS_OFFLINE=1 python main.py
```

Or run individual model scripts directly:

```bash
HF_HUB_OFFLINE=1 TRANSFORMERS_OFFLINE=1 python src/transformers_pipeline/run_mbert.py
HF_HUB_OFFLINE=1 TRANSFORMERS_OFFLINE=1 python src/transformers_pipeline/run_xlmr.py
```

## Configuration

### Baseline hyperparameters — `src/baseline/model.py`
```python
TFIDF_CONFIG  = {"max_features": 5000, "ngram_range": (1, 2), "min_df": 2, ...}
LOGREG_CONFIG = {"max_iter": 1000, "class_weight": "balanced", "random_state": 42}
```

### Transformer hyperparameters — `src/transformers_pipeline/transformer_train.py`
```python
NUM_EPOCHS    = 3
BATCH_SIZE    = 8
LEARNING_RATE = 2e-5
MAX_LENGTH    = 128
THRESHOLD     = 0.3   # sigmoid binarisation threshold
POS_WEIGHT_CAP = 10.0 # cap on per-label positive-class weight
SEED          = 42
```

## Reproducibility

All experiments use fixed seeds (`random_state=42`, `seed=42`).
Transformer experiments were run on CPU (Apple Silicon, macOS).
Results may vary slightly across platforms due to floating-point differences
in scikit-learn's L-BFGS solver and PyTorch CPU kernels.

## Troubleshooting

| Problem | Fix |
|---|---|
| `ModuleNotFoundError: No module named 'src'` | Run from project root: `python main.py` |
| Dataset not found | Check `data/BRIGHTER/{zul,xho,swa}/` folder names and parquet filenames |
| Transformer produces all-zero predictions | Increase epochs to ≥3, lower threshold to 0.3, ensure `pos_weight` is active |
| CUDA / memory errors | Reduce `BATCH_SIZE` in `transformer_train.py` |
| HuggingFace download errors | Use `HF_HUB_OFFLINE=1 TRANSFORMERS_OFFLINE=1` once models are cached |
